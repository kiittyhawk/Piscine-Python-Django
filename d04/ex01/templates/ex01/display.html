Display process of a static page

A static website contains a set of relevant HTML pages and files hosted on the computer on which the web server is installed.

All HTML code is created by the developer until the moment when the page is placed on the server. Since the HTML code does not change after the page is placed on the server, this page is called static.

A web server is software that provides web pages in response to requests from web browsers. Usually, a page request is created when clicking a link on a web page, selecting a bookmark in the browser, or entering a URL in the browser's address bar. The final content of a static web page is determined by the developer and remains unchanged during the page request process.

  1. The web browser requests a static page.

  2. The web server finds the page.

  3. The web server sends the page to the browser that requested it.

Note. If you approach strictly, a "static" page may not actually be one. For example, a replacement image or Flash content (in the form of a SWF file) allows you to "revive" a static page. However, in this context, the page is static, since it is sent to the browser without changes.
